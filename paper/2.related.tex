\section{Related Work}
\label{sec:related}

This section reviews prior work in symbolic regression for trading, organized along two dimensions: (1) the search paradigm (evolutionary vs. neural-guided), and (2) the optimization objective (accuracy vs. profitability).

%------------------------------------------------------------------------------
\subsection{Symbolic Regression: Evolutionary vs. Neural-Guided}

\textbf{Evolutionary Approaches.} Genetic Programming (GP)~\cite{koza1994genetic} has been the dominant paradigm for symbolic regression since the 1990s, evolving expression trees through selection, crossover, and mutation. Schmidt and Lipson~\cite{schmidt2009distilling} demonstrated GP's ability to rediscover physical laws. Modern implementations like gplearn~\cite{stephens2016gplearn} and Operon~\cite{burlacu2020operon} improve efficiency through vectorized evaluation. However, GP suffers from bloat (uncontrolled growth) and premature convergence on rugged fitness landscapes~\cite{poli2008field}.

\textbf{Neural-Guided Approaches.} Recent work leverages neural networks to guide symbolic search:
\begin{itemize}
    \item \textbf{DSR}~\cite{petersen2021deep}: Uses an RNN as a policy network trained via REINFORCE to generate expressions token-by-token.
    \item \textbf{AI-Feynman}~\cite{udrescu2020ai}: Combines neural fitting with physics-inspired techniques (dimensional analysis, symmetry detection).
    \item \textbf{Transformer-based}~\cite{biggio2021neural}: Maps input-output pairs directly to symbolic expressions.
\end{itemize}

%------------------------------------------------------------------------------
\subsection{Monte Carlo Tree Search for Symbolic Regression}

Monte Carlo Tree Search (MCTS)~\cite{coulom2006efficient} has emerged as a powerful alternative to GP, balancing exploration and exploitation through the UCB formula~\cite{kocsis2006bandit}. Key milestones include White et al.~\cite{white2015programming} applying MCTS to symbolic regression and Cazenave~\cite{cazenave2013monte} introducing nested MCTS.

Most relevant to our work is \textbf{NEMoTS} (Neural Equation Monte Carlo Tree Search)~\cite{sun2022symbolic}, which combined MCTS with a Policy-Value network inspired by AlphaGo~\cite{silver2016mastering}. NEMoTS achieved state-of-the-art results on standard SR benchmarks.

\textbf{EATA vs. NEMoTS.} While EATA builds upon the NEMoTS framework, we introduce three critical innovations tailored for financial trading:
\begin{enumerate}
    \item \textbf{Profit Head}: NEMoTS uses policy and value heads for search guidance and expression quality estimation (MSE-based). EATA adds a \textit{third} head explicitly predicting trading profitability via Wasserstein distance.
    \item \textbf{Domain Grammar}: NEMoTS uses generic mathematical operators. EATA introduces domain-specific financial operators (e.g., MA, RSI, Momentum) with fixed windows.
    \item \textbf{Sliding Window}: EATA explicitly addresses non-stationarity through temporal adaptation and expression inheritance, whereas NEMoTS assumes a stationary data distribution.
\end{enumerate}

\subsection{Machine Learning for Algorithmic Trading}

Deep learning approaches for trading include LSTM/GRU~\cite{fischer2018deep,chen2015gru} for capturing long-term dependencies and Transformers~\cite{ding2020hierarchical} for multi-scale pattern detection. Deep Reinforcement Learning (DRL) frameworks like FinRL~\cite{liu2020finrl} frame trading as sequential decision-making. While these methods achieve strong performance, they remain black-boxes, limiting regulatory compliance and human oversight~\cite{cao2022ai}.
